━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SPFレコードは1ドメインに1つしか持てない仕様のため、
既存の社員メール用SPFを **絶対に上書きしないでください**。
Resend用の include を **追記する形** でお願いします。

  ❌ NG（既存を上書き）：
     v=spf1 include:_spf.resend.com ~all

  ✅ OK（既存に追記）：
     v=spf1 include:既存のもの include:_spf.resend.com ~all

万一上書きすると社員メールの到達性が落ちる恐れがありますので、
ここだけ慎重にお願いします。

## 🛠️ ステップ3：Resend ダッシュボードでの操作手順（田代さん用メモ）

システム部から「方針OK、具体的なレコード値ください」と返事が来たら、以下を実行します。

### 3-1. Resend にドメイン追加

1. https://resend.com/domains にアクセス
2. **「Add Domain」** をクリック
3. ドメイン入力：`ielove-partners.jp`
4. リージョン：**Tokyo (ap-northeast-1)** を選択
5. 「Add」をクリック
6. 表示されるレコード（SPF/DKIM/DMARC）を**全てスクショ + テキストコピー**
7. ステップ2のテンプレに値を貼り付けてシステム部に送信

### 3-2. システム部から設定完了の連絡が来たら

1. https://resend.com/domains を再度開く
2. 該当ドメイン横の **「Verify DNS Records」** をクリック
3. 全てのレコードが ✅ になることを確認（数分〜30分かかる場合あり）
4. ステータスが **"Verified"** になれば成功 🎉

## ⚠️ トラブル時の対応

### Resend の Verify ボタンが赤いまま緑にならない
- DNS伝播待ち（最大24時間、通常30分〜1時間）
- システム部に「設定済みのレコード値を再確認してほしい」と依頼
- 特に SPF の include 構文が間違っていないか確認

### 送信したが届かない
- Render のログを確認：https://dashboard.render.com → kaketsuke-form-web → Logs
- 受信側の迷惑メールフォルダを確認
- DMARC レコードを `p=none;` に設定（既設で `p=reject` の場合は到達しないことがある）

### 社員メールが届かなくなった（最悪ケース）
- **すぐにシステム部に連絡** → SPF レコードを元に戻してもらう
- Resend 側を一旦無効化（サブドメイン方式に切替）
