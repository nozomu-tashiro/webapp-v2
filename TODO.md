## 🛡 開発ルール（厳守事項）

`PROJECT_NOTES.md` の方針を踏襲：

### ❌ 絶対NG
1. **webapp から webapp-v2 へのコピー** → webapp は古いコードの可能性
2. **webapp で直接修正** → webapp-v2 との不整合
3. **ApplicationForm.js の安易な置き換え** → 機能消失リスク
4. **git reset --hard の多用** → 必ず現状確認してから
5. **vercel.json の安易な追加/削除** → 過去に何度も混乱の原因

### ✅ 正しいフロー
1. **webapp-v2 で開発**（フロント・バック両方）
2. **バックエンド修正時は kaketsuke-form-web にもコピー**してRender反映
3. **作業ブランチ → PR → main マージ** を厳守
4. **大規模変更前にユーザー確認**
